# 성진프린트 홈페이지 구조 재설계 논의

> 이 문서는 사이트 전체 구조 재설계에 대한 논의 내용을 정리한 것입니다.
> 어떤 AI에든 이 문서를 넣으면 바로 이어서 작업할 수 있도록 작성되었습니다.

---

## 1. 우리는 누구인가

- **성진프린트** — 온라인 인쇄 주문 시스템 (프린트샵)
- 고객이 용지, 사이즈, 인쇄 옵션을 선택하고 파일 업로드 후 주문
- 기술 스택: Astro 5 SSR + React 19 + Supabase + Tailwind CSS 4 + Vercel

---

## 2. 현재 구조 (AS-IS) 와 문제점

### 헤더 메뉴

```
회사소개 | 서비스 | Works | Prints | Edu+100 | 공지사항 | 문의하기
```

### 페이지별 역할과 문제

| 페이지   | URL             | 현재 역할                     | 문제점                                                     |
| -------- | --------------- | ----------------------------- | ---------------------------------------------------------- |
| 서비스   | `/services`     | 서비스 소개 (UI 껍데기)       | Prints 상품과 내용 중복. UI만 살려놓은 상태                |
| Works    | `/works`        | 포트폴리오                    | Prints를 복사해서 수정하려고 남겨놓은 것. 실질적 활용 없음 |
| Prints   | `/prints`       | 포트폴리오형 갤러리           | `linked_product_id`로 상품 연결 가능. 실제 활용 중         |
| Products | `/product/[id]` | 주문 빌더 (커스터마이징+결제) | 고객에게 직접 노출되는 메뉴 없음                           |
| Edu+100  | `/edu100`       | 교재 표지 디자인 갤러리       | 단독 개념. 윤전제본 외주 전용                              |
| Blog     | `/blog/[slug]`  | 블로그 포스트                 | edu100 섹션에 종속 (404시 /edu100으로 리다이렉트)          |
| News     | `/news`         | 공지사항                      | 정상 운영 중                                               |

### 핵심 혼란: 3중 구조

고객이 "무선제본 주문하려면 어디를 클릭해야 하지?"

- 서비스? → UI만 있고 빈 껍데기
- Prints? → 소개글은 있는데 주문 버튼이 있을 수도 없을 수도
- Products? → 메뉴에 노출 안 됨

## 3. 결정된 구조 (TO-BE)

### 핵심 개념: 2 레이어

```
Layer 1: 콘텐츠 (왜 이 상품이 좋은지) → Prints 소개글
Layer 2: 주문 (실제 커스터마이징+결제) → Product 빌더
```

- **Prints** = 상품 소개 콘텐츠 허브 (블로그처럼 읽는 곳)
- **Products** = 주문 빌더 (Prints에서 "주문하기"로 진입)
- 고객은 Prints를 읽고 → 주문하기 클릭 → Product 빌더로 이동

### 헤더 메뉴 (TO-BE)

```
회사소개 | 인쇄 서비스(→/prints) | Edu+100 | 공지사항 | 문의하기
```

### 각 영역의 운명

| 현재                       | 결정                        | 이유                                                 |
| -------------------------- | --------------------------- | ---------------------------------------------------- |
| 서비스 (`/services`)       | **삭제 또는 prints로 통합** | UI만 있고 중복                                       |
| Works (`/works`)           | **삭제**                    | prints 복사본. DB 테이블도 정리 대상                 |
| Prints (`/prints`)         | **메인 상품 허브로 승격**   | 이미 `linked_product_id`로 빌더 연결 가능            |
| Products (`/product/[id]`) | **유지 (빌더 전용)**        | 메뉴 직접 노출 안 함. prints에서 진입                |
| Blog (`/blog/[slug]`)      | **edu100 전용 유지**        | 필요시 prints 콘텐츠로 확장 가능                     |
| Edu+100                    | **단독 유지**               | 윤전제본 외주 디자인갤러리. 다른 상품과 섞이면 안 됨 |

### 고객 동선 (User Flow)

```
랜딩 (/)
├─ "인쇄 서비스" → /prints (상품 갤러리)
│   └─ /prints/[id] (무선제본 소개글)
│       └─ "주문하기" → /product/[id] (빌더)
│           └─ 옵션 선택 → /upload → /checkout → /order-complete
│
├─ "Edu+100" → /edu100 (디자인 갤러리)
│   └─ 커버 클릭 → 모달
│       └─ "이 디자인으로 주문" → /product/[id]?designId=xxx
│           └─ 빌더 → /upload → /checkout
│
└─ 랜딩 섹션에서 직접 진입
    ├─ 인쇄 서비스 소개 → /prints
    ├─ Edu+100 소개 → /edu100
    └─ CTA → /contact
```

## 4. 랜딩페이지 구조 (TO-BE)

### 랜딩페이지의 성격

- **가벼운 세일즈형 (Soft B)** — 강한 세일즈는 아니지만 "여기 괜찮은 곳이구나" 신뢰를 주는 흐름
- 스토리라인: "우리 이런 곳이에요" → "이런 거 해요" → "이렇게 같이 일해요" → "관심 있으면 여기로"

### 섹션 구성 (확정)

```
┌─────────────────────────────────────────────────┐
│ 1. Hero                                         │
│    CMS 관리 타이틀 + 히어로 이미지               │
│    (CTA 버튼 추가 검토 필요)                     │
├─────────────────────────────────────────────────┤
│ 2. Partners                                     │
│    "다양한 업종의 고객사와 함께하고 있습니다"      │
│    로고 무한 슬라이드 (그레이→컬러 hover)         │
├─────────────────────────────────────────────────┤
│ 3. 인쇄 서비스 소개                              │
│    대표 이미지 + 짧은 설명                        │
│    "인쇄 서비스 보기 →" 링크 → /prints           │
│    (상품 5개를 직접 나열하지 않음 — 카탈로그 느낌 방지) │
├─────────────────────────────────────────────────┤
│ 4. 이렇게 일합니다 ★ 신규 섹션                   │
│    3개 카드 형태:                                │
│                                                 │
│    [업체 파트너십]     [디자인 동반자]    [장기 교재 파트너] │
│    납기를 맞추는       컨셉부터 인쇄까지  10년, 50종       │
│    신뢰 관계          함께 진행         교재를 함께       │
│                                                 │
│    → 업체들과 시간 맞춰주는 관계                  │
│    → 개인/학원 선생님과 디자인·컨셉·기획 장기 진행 │
│    → 대형 유치원과 10년째 교재 50종 디자인+인쇄   │
├─────────────────────────────────────────────────┤
│ 5. Edu+100 소개                                 │
│    "교재 표지 디자인 갤러리"                      │
│    커버 미리보기 3-4개                            │
│    "갤러리 보기 →" 링크 → /edu100               │
├─────────────────────────────────────────────────┤
│ 6. 뉴스                                        │
│    최신 공지 2-3개                               │
├─────────────────────────────────────────────────┤
│ 7. CTA                                         │
│    "문의하기" / "견적 상담"                       │
└─────────────────────────────────────────────────┘
```

### 기존 섹션 → 새 섹션 매핑

| 기존                        | 새 구조             | 변경 사항                                             |
| --------------------------- | ------------------- | ----------------------------------------------------- |
| Hero (HeroDB.astro)         | Hero                | 유지. CTA 버튼 추가 검토                              |
| Partners (PartnersDB.astro) | Partners            | 유지 (그대로)                                         |
| Service (Service.astro)     | 인쇄 서비스 소개    | 상품 직접 나열 → 소개+링크 형태로 변경                |
| Works (WorksDB.astro)       | **삭제**            | "이렇게 일합니다"로 대체                              |
| About (About.astro)         | **삭제 (랜딩에서)** | /about 페이지에만 유지. 또는 "이렇게 일합니다"에 흡수 |
| News (News.astro)           | 뉴스                | 유지                                                  |
| CTA (CTA.astro)             | CTA                 | 유지                                                  |
| (없음)                      | **이렇게 일합니다** | 신규 섹션                                             |
| (없음)                      | **Edu+100 소개**    | 신규 섹션 (랜딩용 하이라이트)                         |

### 랜딩 디자인 원칙

- **사진 연속 자체는 문제 아님** (애플도 사진 연속). 문제는 "다 똑같아 보이는 것"
- 섹션마다 **레이아웃, 배경색, 크기**를 다르게 해서 시각적 변화 줄 것
- 랜딩은 **각 섹션의 맛보기**만. 깊은 내용은 해당 페이지에서 (/prints, /edu100)

## 5. "이렇게 일합니다" 섹션 상세

### 이 섹션이 필요한 이유

성진프린트의 핵심 강점은 **관계 기반 장기 파트너십**:

1. **업체 파트너십** — 업체들과 시간(납기)을 맞춰주는 신뢰 관계
2. **디자인 동반자** — 개인/학원 선생님과 디자인·컨셉·기획을 오랫동안 도맡아 진행
3. **장기 교재 파트너** — 동네 대형 유치원과 10년째 교재 50종을 디자인+인쇄

### 톤

- 과하지 않게. 카드 3개 + 짧은 카피 정도
- 사진보다는 **아이콘 + 텍스트** 형태가 적합 (리듬 변화 + 가독성)
- 구체적 숫자가 있으면 신뢰감 높아짐 ("10년", "50종")

## 6. Edu+100 독립 개념 정리

- **윤전제본 전용** — 다른 상품(무선/중철/스프링 등)과 섞이지 않는 독립 서비스
- **외주 인쇄** — outsourced 상품 타입 사용
- **디자인 갤러리** — 표지 디자인을 갤러리에서 선택 → 주문 연결
- DB: `edu100_sections` (섹션 관리) + `edu100_covers` (표지 데이터) + `blog_posts` (블로그)
- 랜딩에서는 **하이라이트 미리보기**만 노출, 풀 갤러리는 `/edu100`에서
